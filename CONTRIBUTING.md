# Contributing to MirageAPI

Thank you for your interest in contributing to MirageAPI! We welcome contributions from the community.

## How to Contribute

### Reporting Issues
- Use GitHub Issues to report bugs or suggest features
- Provide clear descriptions and reproduction steps
- Include environment details (Node.js version, OS, etc.)

### Code Contributions
1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-feature-name`
3. Make your changes with clear commit messages
4. Add tests for new functionality
5. Ensure all tests pass: `yarn test`
6. Submit a pull request

### Development Setup
```bash
# Clone your fork
git clone https://github.com/yourusername/mirage.git
cd mirage

# Install dependencies
yarn install

# Start development server
yarn dev

# Run tests
yarn test

# Start production server
yarn start:prod
```

## Code Style
- Follow existing code patterns and conventions
- Use meaningful variable and function names
- Add comments for complex logic
- Keep functions focused and concise
